СТАУСЫ ФАЙЛОВ И ОСНОВНЫЕ КОМАНДЫ GIT

Созданный файл имеет статус "untracked"
Измененный файл имеет статус "modified"

Стандартный флоу пуша файлов в репозиторий GitHub:
1. Создать проект
2. Создать репозиторий на GitHub
   2.1. git clone *адрес репозитория*        # можно клонировать репозиторий на ПК (тогда придется потом каждый раз активировать вирт.окр.)
   2.2. python3 -m venv venv                 # создать вирт. окружение (если требуется)
   2.3. source venv/bin/activate             # активировать вирт. окружение (или venv\Scripts\activate)
3. git add . или git add *имя файла/папки*   # подготовить добавленные/измененные файлы к коммиту, сделать их отслеживаемыми ("staged")
4. git commit -m "*сообщение*"               # сделать очередной коммит
5. git push                                  # сделать пуш снимка локального репозитория в глобальный репозиторий

Дополнительные команды:
git status                          # узнать текущий статус репозитория
git log                             # отображает все коммиты в обратном хронол. порядке (свежие коммиты будут вверху)
git init                            # инициализировать новый репозиторий Git с началом отслеживания текущей папки (альт. способ п.2)
