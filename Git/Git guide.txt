СТАУСЫ ФАЙЛОВ И ОСНОВНЫЕ КОМАНДЫ GIT

Созданный файл имеет статус "untracked"
Измененный файл имеет статус "modified"

Стандартный флоу пуша файлов в репозиторий GitHub (с двумя альт. способами начала работы с Git: п.1.1. или п.1.2.):
1. Создать репозиторий на GitHub
2. git clone *адрес репозитория*    # склонировать по ссылке репозиторий из GitHub на локальный ПК
3. cd *имя репозитория*             # перейти в его папку
4. python3 -m venv venv             # создать вирт. окружение (если требуется)
5. source venv/bin/activate         # активировать вирт. окружение (опять же, если требуется)
6. git commit -m "сообщение"        # сделать первый коммит - снимок/фиксацию файлов в локальный репозиторий ("commited")
7. git add . или git add *имя файла/папки*   # подготовить добавленные/измененные файлы к коммиту, сделать их отслеживаемыми ("staged")
8. git commit -m "сообщение"        # сделать очередной коммит
9. git push                         # сделать пуш снимка локального репозитория в глобальный репозиторий

Дополнительные команды:
git status                          # узнать текущий статус репозитория
git log                             # отображает все коммиты в обратном хронол. порядке (свежие коммиты будут вверху)
git init                            # инициализировать новый репозиторий Git с началом отслеживания текущей папки (альт. способ п.2)
